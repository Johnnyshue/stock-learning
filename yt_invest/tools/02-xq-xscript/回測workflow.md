# XQ 回測 Workflow（誠實版流程說明）

> ⚠️ **重要誠實聲明**：回測必須由 **XQ全球贏家軟體本身** 執行。
> 本工具包只能幫你「生成 XScript 腳本」與「整理回測流程/比較表」，
> **無法代你跑回測**——因為回測要讀 XQ 的歷史資料庫並用 XQ 引擎運算，
> 那是裝在你電腦上的軟體，Claude 沒有它的執行權限。
> 影片裡「睡覺時 AI 跑回測」指的是有人另外做了一套自動化（把 Claude 接上 XQ 操作），
> 那需要額外的橋接工具，不在這個純腳本工具包範圍內。**這裡誠實標清楚，不誇大。**

---

## 一、基本流程（手動，最可靠）

```
自然語言條件
   │  （用 SKILL.md 生成）
   ▼
XScript 選股腳本（.txt）
   │  複製全文
   ▼
XQ →【選股中心 / 編輯器】貼上
   │
   ▼
按【編譯】  ←──────────────┐
   │                        │  紅字報錯？
   ├─ 通過 ✅               │  把錯誤訊息貼回給 Claude，
   │                        │  請它對照修正函數名 ──┘
   ▼
【加入選股】→ 執行 → 篩出符合條件的股票清單
```

### 編譯報錯時怎麼辦（高機率會遇到）
因為腳本是 best-effort，最常見的錯是「函數名/欄位名不存在」。處理方式：
1. 把 XQ 的紅字錯誤訊息（含行號）**整段貼回給 Claude**。
2. Claude 對照訊息，把該行的 `// TODO` 函數名改成正確的。
3. 重新編譯，重複到綠燈。
4. **這個來回是正常的，不是失敗**——這就是為什麼 SKILL.md 堅持標 TODO。

## 二、把選股策略變成「可回測的交易策略」

選股腳本（篩當下符合的股票）≠ 回測（驗證歷史績效）。要回測得：
1. 把選股條件改寫成 **交易策略腳本**（XQ 的策略模組，含進場 `SetPosition`/`Buy`、
   出場條件、停損停利）。
   - ⚠️ 交易策略的進出場語法（Buy/Sell/SetPosition 等）我同樣 **無法保證正確**，
     生成時會照 SKILL.md 規則標 TODO，請以 XQ 編譯為準。
2. 在 XQ 的**策略回測**模組載入該策略，設定回測期間、初始資金、手續費。
3. 跑出績效報告。

## 三、影片講的「睡覺跑多組因子回測，早上看報告」是什麼

影片描述的早上報告，核心是比較**多組策略**的這幾個指標：

| 指標 | 意義 | 怎麼用 |
|---|---|---|
| 獲利因子 (Profit Factor) | 總獲利 ÷ 總虧損 | > 1 才賺；越高越好 |
| 勝率 (Win Rate) | 賺錢交易 ÷ 總交易數 | 高勝率 ≠ 高報酬，要配合賠率看 |
| 最大回撤 (Max Drawdown) | 從高點到低點最大跌幅 | 越小代表越抗震，影響你睡不睡得著 |
| 適合盤勢 | 多頭/空頭/盤整哪種環境有效 | 對照當前 VIX/趨勢挑策略 |

**這份比較報告 Claude 可以幫你做**——但前提是**你把 XQ 跑出來的每組回測數字貼給我**，
我幫你整理成對照表、找出獲利因子最高/回撤最小的那組。
**我無法自己生出這些數字**（那要 XQ 引擎跑歷史資料）。給我真實數字 → 我整理；
沒有數字 → 我不會編造績效，這違反禁止造假鐵則。

### 建議的「多因子比較表」模板（你填數字，或貼給 Claude 整理）

| 策略 | 回測期間 | 年化報酬 | 獲利因子 | 勝率 | 最大回撤 | 交易次數 | 備註/適合盤勢 |

| 01 恐慌錯殺價值股 | | | | | | | |
| 02 站上季線+營收轉正 | | | | | | | |

## 四、回測注意事項（避免自欺）

- **前視偏誤 (Look-ahead bias)**：基本面欄位（營收、殖利率）有公布時間差，
  回測若用到「未來才知道的數字」會高估績效。XQ 是否自動處理 point-in-time 請自行確認。
- **倖存者偏誤 (Survivorship bias)**：下市股票若不在資料庫，回測會偏樂觀。
- **過度最佳化 (Overfitting)**：跑很多組因子挑最好的那組，容易是「對歷史過度配適」，
  未必對未來有效。影片那種「跑一堆組合挑最賺」尤其要小心這點。
- **樣本外驗證**：找到好策略後，用沒參與最佳化的期間再驗一次。

> 這些是 best practice 提醒，不是 XQ 專屬功能說明。實際 XQ 怎麼設定請看官方教學或實測。
